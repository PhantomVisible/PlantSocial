"Merci [Prénom]. Bonjour à tous.

 Avant de vous montrer les fonctionnalités, laissez-moi vous expliquer les bases
 techniques sur lesquelles PlantSocial est construit.

 Nous avons choisi une architecture Monolithique Modulaire. Ce n'est pas une
 architecture microservices — chaque fonctionnalité est dans une même application
 mais organisée en modules parfaitement séparés, comme des pièces dans une maison
 bien rangée.

 Pour le Backend, nous utilisons Java avec Spring Boot — une technologie robuste,
 sécurisée et très utilisée en entreprise.

 Pour le Frontend, Angular — un framework moderne qui nous offre une navigation
 ultra-fluide sans rechargement de page.

 La base de données est PostgreSQL, les communications sont sécurisées par des
 tokens JWT, et nous avons un serveur Docker pour faciliter le déploiement.

 Le flux est simple : Le frontend Angular communique avec notre API Spring Boot,
 qui elle-même accède à la base de données PostgreSQL."

SLIDE 6 : BACKEND DÉTAILLÉ — Spring Boot
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[ Personne 2 — VOUS ]

"Allons maintenant plus loin dans le backend.

 Notre backend Spring Boot est organisé en couches claires :

 1. Sécurité : toutes les requêtes passent d'abord par un filtre Spring Security qui
    vérifie le token JWT. Aucun accès non autorisé n'est possible.

 2. Les REST Controllers : chaque domaine a ses propres endpoints. Par exemple :
    /marketplace pour les annonces, /chat pour les messages, /plants pour le jardin IA.

 3. La couche Service : c'est là que vit toute la logique métier. Chaque fonctionnalité
    a son propre service Spring, ce qui rend le code propre et maintenable.

 4. JPA et les Repositories : nous accédons à PostgreSQL via Spring Data JPA.
    Les requêtes complexes sont gérées avec l'annotation @Query.

 5. WebSocket avec le protocole STOMP : c'est ce qui permet le chat en temps réel.
    STOMP gère les abonnements aux topics de messages côté serveur.

 6. Les APIs externes : nous appelons l'API Pl@ntNet pour identifier les plantes, et
    une NewsAPI pour afficher les actualités. Ces appels se font via RestTemplate
    depuis notre backend Spring Boot."

"Côté frontend Angular, nous avons appliqué les meilleures pratiques du framework.

 L'application est divisée en Feature Modules : chaque grande fonctionnalité
 (l'authentification, le marketplace, le chat, le profil) est dans son propre module
 Angular, chargé à la demande. Cela améliore les performances au démarrage.

 Pour les données asynchrones, nous utilisons RxJS et les Observables. Quand le
 frontend fait une requête à l'API, il s'abonne à l'Observable et met à jour
 l'interface automatiquement à la réception des données.

 La sécurité côté client est gérée par les Guards : si un utilisateur non connecté
 essaie d'accéder au marketplace, il est automatiquement redirigé vers le login.
 Et un Interceptor HTTP injecte le token JWT dans chaque requête sortante.

 Pour le style, nous avons un système de thème complet basé sur des variables CSS.
 En changeant quelques variables, toute l'interface bascule du mode clair au mode
 sombre.

 Pour le chat temps réel côté client, nous utilisons la bibliothèque SockJS avec
 le protocole STOMP pour se connecter au serveur WebSocket et recevoir les messages
 en temps réel.

 Et pour les composants UI, nous utilisons PrimeNG, une bibliothèque de composants
 Angular que nous avons entièrement personnalisée avec notre propre SCSS."

"Parlons maintenant du défi technique majeur que j'ai personnellement relevé :
 le temps réel.

 Un réseau social sans instantanéité, c'est comme un téléphone sans réseau.
 Ça ne fonctionne pas.

 Pour comprendre pourquoi les WebSockets sont révolutionnaires, voici une analogie
 simple : imaginez que vous attendez un colis. Avec HTTP classique, vous devez appeler
 le livreur vous-même toutes les 5 minutes pour demander 'est-ce que mon colis est
 là ?' C'est lent et inutile. Avec WebSocket, le livreur vous appelle directement dès
 qu'il arrive devant votre porte.

 C'est exactement ce mécanisme que j'ai implémenté. Notre serveur Spring Boot maintient
 une connexion permanente ouverte avec chaque utilisateur connecté. Dès qu'un message
 est envoyé, il est diffusé instantanément au destinataire — sans délai, sans
 rechargement.

 J'ai également développé les Notifications Push : quand quelqu'un commente votre
 post ou vous envoie un message, une alerte apparaît immédiatement dans l'interface,
 même si vous êtes sur une autre page de l'application."

"Merci [Prénom]. L'une des fonctionnalités dont nous sommes les plus fiers est
 'Mon Jardin', et c'est une vraie intégration d'Intelligence Artificielle — pas un
 projet futur, c'est déjà en production dans notre application.

 Voici comment ça fonctionne en 4 étapes simples :

 Étape 1 : L'utilisateur prend en photo une plante avec son téléphone et l'uploade
 dans l'application.

 Étape 2 : Notre backend Spring Boot envoie cette image à l'API Pl@ntNet. Pl@ntNet
 est une intelligence artificielle botanique développée par des chercheurs français,
 capable d'identifier plus de 300 000 espèces de plantes dans le monde.

 Étape 3 : L'API nous renvoie en quelques secondes le nom scientifique et le nom
 commun de la plante.

 Étape 4 : La plante identifiée est automatiquement sauvegardée dans le profil de
 l'utilisateur, dans sa collection personnelle 'Mon Jardin'.

 L'avantage ? L'utilisateur n'a besoin de rien connaître en botanique. Il pointe,
 il photographie et l'IA fait le reste."

"Notre deuxième fonctionnalité IA s'appelle 'Doctor Plant', et c'est là que
 PlantSocial devient vraiment unique.

 Le problème que cette fonctionnalité résout est très concret : votre plante a les
 feuilles qui jaunissent, ou des taches brunes, et vous ne savez pas pourquoi.
 Avant, vous pouviez poster une photo sur un forum et attendre des heures une réponse.
 Avec Doctor Plant, l'IA vous répond en quelques secondes.

 Voici le parcours :

 L'utilisateur uploade une photo de sa plante malade. Notre système envoie l'image à
 un modèle d'intelligence artificielle de vision qui analyse les symptômes visuels :
 jaunissement des feuilles, pourriture des racines, présence de parasites, taches
 caracteristiques de maladies fongiques...

 L'application génère ensuite un rapport de santé complet : le diagnostic du problème,
 le niveau de gravité, et les recommandations de traitement.

 Ce qui rend cette fonctionnalité différenciante, c'est qu'elle est impossible à
 trouver sur n'importe quelle plateforme généraliste. C'est une valeur ajoutée
 exclusive à PlantSocial."

QUESTIONS TYPES DU JURY — RÉPONSES PRÊTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q1 : "Quelle différence avec Facebook ou Instagram ?"
R : "Ces plateformes sont généralistes. Sur PlantSocial, TOUT est conçu pour
     les plantes : les profils montrent les collections, les annonces sont liées
     aux vendeurs passionnés, et les fonctionnalités IA (My Garden, Doctor Plant)
     sont impossibles à trouver ailleurs. C'est la différence entre un supermarché
     et une épicerie fine spécialisée."

Q2 : "Pourquoi une architecture monolithique et non microservices ?"
R : "Pour un projet de cette échelle et à ce stade, le monolithe modulaire est
     le meilleur choix : déploiement simple, maintenance facile, performance
     optimale. Les microservices ont du sens quand une équipe de 30+ développeurs
     travaille sur des centaines de fonctionnalités. Pour notre contexte, c'était
     sur-ingénierie."

Q3 : "Comment sécurisez-vous les paiements ?"
R : "Pour cette version, le paiement est simulé. Dans une version production, nous
     intègrerons Stripe qui gère toute la sécurité PCI DSS. Notre rôle serait de
     gérer les webhooks Stripe et de mettre à jour les statuts des annonces."

Q4 : "Pl@ntNet est-elle une IA que vous avez entraînée vous-mêmes ?"
R : "Non, et c'est un choix délibéré. Pl@ntNet est une IA botanique académique
     développée par l'INRIA, ayant analysé des millions de photos de plantes pendant
     des années. Utiliser cette API nous permet d'offrir une précision d'identification
     que nous aurions mis des années à développer seuls. C'est l'approche 'API-first',
     standard dans l'industrie."

Q5 : "Comment acquérir les premiers utilisateurs ?"
R : "Par la stratégie de niche. Nous contactons les créateurs de contenu jardinage
     sur Instagram et TikTok (les 'Plantfluencers') pour en faire des ambassadeurs.
     Nous ciblons aussi les groupes Facebook d'échange de boutures existants.
     Ces utilisateurs sont déjà passionnés et cherchent exactement ce que nous offrons."
