03 — Technical Architecture

Stack & Architecture

Overview

PlantSocial is built on a Modular Monolith Architecture — a single well-structured application that is robust, easy to maintain, and ready to scale.

  • ☕
    Backend — Spring Boot: Enables a structured REST API + real-time WebSocket endpoint, organized into fully independent feature modules
  • 🅰️
    Frontend — Angular: Delivers a seamless Single Page Application with dynamic routing, lazy-loading, and strict TypeScript typing
  • 🐘
    Database — PostgreSQL: Handles complex relational data (users ↔ posts ↔ listings ↔ plants) with ACID transaction guarantees
  • 🔐
    Security — JWT: Provides stateless, scalable authentication — no server-side sessions, every request is self-verified
  • 🔌
    Real-time — WebSockets: Powers the live chat and notification system via STOMP protocol — zero polling, zero page reload
  • 🐳
    Deployment — Docker: Containerizes the backend and database, ensuring a consistent environment across all developer machines
Communication Flow: 📱 Angular SPA → ☕ Spring Boot API → 🐘 PostgreSQL
💡
Why not Microservices? For a 4-developer project, a modular monolith provides the same separation of concerns without the operational overhead of managing multiple independent services.
🌍
External APIs Integrated Pl@ntNet (AI plant identification) and NewsAPI (botanical news feed) are consumed directly from the Spring Boot backend.
03 — Architecture: Backend

Backend — Spring Boot

Layered Architecture

The backend follows the Controller → Service → Repository pattern — each layer has a single responsibility, making the code clean, testable, and easy to extend.

  • 🔐
    JWT Security: Every request passes through a Spring Security filter — access is denied unless a valid token is provided
  • 🌐
    REST Controllers: Domain-specific endpoints — /auth, /users, /marketplace, /chat, /feed, /plants
  • ⚙️
    Service Layer: All business logic is encapsulated in dedicated Spring Services — one service per feature, fully decoupled
  • 🗄️
    JPA / Repositories: Database access via Spring Data JPA ORM — complex queries handled with custom @Query annotations
  • 🔌
    WebSocket STOMP: Dedicated endpoint manages topic subscriptions and broadcasts real-time events to connected clients instantly
  • 🌍
    External API Calls: RestTemplate / WebClient connects to Pl@ntNet and NewsAPI — integrated directly in the service layer
🔐
Multi-layer Security Request flow: JWT Filter → Spring Security → Controller → Service. USER / ADMIN roles enforced via @PreAuthorize on sensitive endpoints.
🔌
WebSocket + Online Presence A PresenceService listens to WebSocket connect/disconnect events and broadcasts the live "online" status to all clients via /topic/presence.
🧩
19 Controllers — 22 Services Auth, Feed, Post, Comment, Follow, Chat, Notification, Marketplace, Shop, Plant, PlantId, PlantDoctor, News, Trend, Block, Report…
03 — Architecture: Frontend

Frontend — Angular

Modular SPA Application

The Angular application is split into independent Feature Modules, each loaded on demand. Every module owns its components, services, and routes — fully self-contained.

  • 📦
    Feature Modules: Auth, Feed, Marketplace, Chat, Profile, Garden, PlantDoctor, News, Shop — isolated modules, lazy-loaded for faster startup
  • 🔄
    Services & RxJS Observables: Asynchronous API data is managed via HttpClient + RxJS Observables and the async pipe for reactive UI updates
  • 🛡️
    Guards & Interceptors: AuthGuard blocks private routes — HttpInterceptor automatically attaches the JWT Bearer token to every outgoing HTTP request
  • ⚡
    WebSocket Client (STOMP/SockJS): Persistent connection managed by a dedicated service — subscribes to topics and updates the UI in real time without any page reload
  • 🎨
    SCSS Theme System: Dynamic CSS variables enable instant light ↔ dark mode switching across the entire UI without a single reload
  • 🧩
    PrimeNG UI Library: Enterprise-grade UI components fully customized with our own SCSS design system for a consistent, professional look
🛡️
Client-side Security Route Guard → redirects to /login if unauthenticated. JWT Interceptor → injects the Bearer token into 100% of outgoing HTTP requests automatically.
⚡
WebSocket Client ChatService + NotificationService maintain a persistent STOMP connection. Every incoming event triggers an immediate UI update — no polling, no reload.
🎨
16 Angular Feature Modules Each major feature is a self-contained module. This improves startup performance via lazy-loading and makes the codebase easier to navigate across a team.
04 — Contribution: Real-Time Chat

Live Chat System

Real-Time Messaging

PlantSocial's chat is built on a persistent WebSocket + STOMP connection — messages are delivered instantly, with no page reload and no polling overhead.

  • 💬
    1-to-1 Live Chat: A permanent bidirectional connection delivers messages the instant they are sent — zero latency, zero page refresh
  • 💬
    Floating Chat Widget: The chat interface overlays any page — users can message without leaving their current view
  • 🟢
    Online Presence Status: A green badge on avatars shows who is currently connected — updated live via WebSocket events
  • 📜
    Conversation History: All messages are persisted in the database and loaded when a conversation is opened — full history always available
🔌 Classic HTTP: The client asks → the server responds. Users must refresh to see new messages.

⚡ WebSocket (our choice): The connection stays always open. The moment a message is sent, the server pushes it directly to the recipient — no request, no delay.
⚙️
Technical Stack Backend: Spring WebSocket + STOMP + PresenceService.
Frontend: SockJS + @stomp/stompjs, subscribed to /user/queue/chat and /topic/presence.
04 — Contribution: Notifications

Real-Time Notifications

Push Notification System

PlantSocial notifies users instantly for every relevant event — a dedicated notification service broadcasts alerts via WebSocket the moment an action happens on the platform.

  • 🔔
    New Message Alert: Notified instantly when someone sends a direct message — includes sender name and message preview
  • ❤️
    Post Like Notification: Triggered the moment another user likes one of your posts — keeping engagement loops alive
  • 💬
    Comment Notification: Instantly notified when someone comments on your post — encouraging real-time conversations
  • 🛒
    Marketplace Interaction: Alerts for activity on your listings — new offers, questions from buyers, or status changes
  • 👨‍🤝‍👨
    New Follower: Notified when someone follows your profile — grows community engagement and awareness
⚙️
How It Works Every action (like, comment, message) triggers the NotificationService in Spring Boot → persists to DB → broadcasts via WebSocket to the target user's personal queue /user/queue/notifications.
🔔
Unread Badge Counter The notification bell in the navbar displays a live unread count, updated in real time. Clicking marks all as read via a REST endpoint.
💾
Persistence + History All notifications are stored in the database. Users can review their full notification history at any time, even after reconnecting.
04 — Contribution: Marketplace

The Integrated Marketplace

End-to-End Listing Platform

PlantSocial's Marketplace enables peer-to-peer plant buying and selling, with a unique trust model rooted in the seller's social profile — making every transaction transparent and community-verified.

  • 📋
    Listing Creation: Sellers publish in seconds — title, description, price, status (Active / Draft / Expired) and a multi-photo gallery
  • 🔗
    Smart URL Import: Paste any URL and the backend's ScraperService auto-extracts the title and preview via Open Graph metadata
  • 🖼️
    Shopify-Style Gallery: Thumbnail strip on the left, full-size image on the right — exactly the professional layout buyers expect from top e-commerce platforms
  • 🛡️
    Trust via Social Profile: Every listing is linked to the seller's public profile — buyers see their posts, their plant collection, and their community reputation before purchasing
  • 💳
    Payment Animation: A 3-step sequential animation (card insert → NFC scan → success screen) delivers a premium checkout experience
  • 🗂️
    Listing Management: Sellers can edit, delete, or update the status of each listing — full control from a dedicated dashboard
🛡️
Trust = Key Differentiator On generic platforms, you buy from a stranger. On PlantSocial, the buyer sees the seller's posts, plant collection, and community activity — building trust before the transaction.
🔗
ScraperService — Smart Import The backend ScraperService fetches Open Graph metadata from any URL and pre-fills the listing form automatically — saving sellers time and reducing manual entry errors.
💳
Realistic Payment Simulation 3-step animation: card insertion → NFC scan → confirmation. Designed to deliver a premium user experience that mirrors a real e-commerce checkout flow.

09 — Business Model & Monetization

How We Fund PlantSocial

Revenue Strategy

To finance PlantSocial, we combine a freemium model with transaction-based fees — keeping the social core completely free to maximize user adoption, while monetizing premium features and marketplace activity.

  • 🆓
    Free Social Core: The feed, profiles, posts, follow system, and chat are permanently free — this drives mass adoption and builds our community base
  • 💰
    Listing Fee: A small commission per listing published on the Marketplace — sellers only pay when active, aligning our interests with theirs
  • 🪴
    My Garden Premium: Unlimited AI plant identifications + full Pl@ntNet database access — unlocked with a monthly subscription for plant enthusiasts
  • 🩺
    Doctor Plant Pro: Unlimited AI health diagnoses + personalized care tracking for a user's full plant collection — billed monthly for power users
  • 🏪
    Pro Seller Account: Dedicated accounts for nurseries and vendors — advanced analytics, boosted listing visibility, and bulk management tools
  • 📣
    Targeted Advertising: A 100% qualified audience of plant lovers = highly relevant ads for fertilizers, pots, UV lamps, and gardening tools
💡
Why Freemium? Free access removes all barriers to entry. Once users are active in the community, converting them to paying subscribers for premium AI features becomes a natural next step.
📈
Growth Flywheel More users → more listings → more transactions → more revenue → better features → even more users. The social layer fuels the marketplace, which fuels the business.
🎯
Target Market Plant hobbyists, indoor gardeners, nurseries, botanical educators, and eco-conscious consumers — a growing, passionate niche with high willingness to pay for quality tools.
